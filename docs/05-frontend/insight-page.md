# Insight 頁面設計

## 概述

Insight 頁面採用「問題導向設計」，直接回答社群經營者的常見問題，而非單純顯示數據。

**設計哲學**：
- 總覽頁（Dashboard）= 數據儀表板，讓用戶自己探索
- Insight 頁 = 問題解答，給出具體結論與建議

---

## 問題清單

Insight 頁面回答 6 大問題：

| # | 問題 | 簡述 | 時間區間 | 資料範圍 | 適用層級 | 狀態 |
|---|------|------|----------|----------|----------|------|
| 1 | 這週表現如何？ | 貼文成效評估 | 7 天（可調：7/14/30 天） | 多篇聚合 | Post → Account | ✅ MVP |
| 2 | 有爆紅跡象嗎？ | 出圈/爆紅偵測 | 近 72 小時內發布 | 單篇篩選 | Post | ✅ MVP |
| 3 | 粉絲在成長嗎？ | 受眾成長追蹤 | 7/30 天趨勢 | 帳號層級 | Account | ✅ MVP |
| 4 | 什麼內容表現最好？ | 內容類型優化 | 30 天（需足夠樣本） | 多篇按類型聚合 | Post → Account | 📌 預留（待標籤功能） |
| 5 | 什麼時候發文最好？ | 最佳發文時段 | 30 天（需足夠樣本） | 多篇按時段聚合 | Post → Account | ✅ MVP |
| 6 | 哪些舊文還有流量？ | 長尾貼文追蹤 | 發布 > 30 天 | 單篇篩選 | Post | ✅ MVP |

### 欄位說明

| 欄位 | 說明 |
|------|------|
| **時間區間** | 這個問題分析的時間範圍 |
| **資料範圍** | 單篇貼文篩選 or 多篇貼文聚合 |
| **適用層級** | Post（貼文層級）/ Account（帳號層級）/ Post → Account（貼文聚合到帳號）|

## 問題詳細設計

### Q1: 這週表現如何？（貼文成效評估）

**回答的核心問題**：整體貼文健康度、值不值得複製成功模式

#### 分析參數

| 參數 | 設定 |
|------|------|
| 時間區間 | 預設 7 天，可調整（7/14/30 天） |
| 資料範圍 | 多篇貼文聚合 |
| 適用層級 | Post → Account |

#### 所需指標

| 指標 | 用途 |
|------|------|
| Engagement Rate（平均） | 整體互動健康度 |
| Reply Rate（平均） | 對話引發力 |
| 個人化基準比較 | 與自己歷史比較 |

#### 呈現方式

```
┌─────────────────────────────────────────┐
│  📊 這週表現如何？                        │
├─────────────────────────────────────────┤
│  整體評價：表現優於平均                    │
│                                         │
│  • 平均 ER：5.8%（好 ✓）                  │
│  • 比上週：+12%                          │
│  • 表現最好：[貼文預覽] ER 12%            │
│  • 表現最差：[貼文預覽] ER 2%             │
│                                         │
│  💡 建議：週三的問答貼文互動最高，可多嘗試  │
└─────────────────────────────────────────┘
```

#### 資料來源

- L3 Current：當前貼文成效
- L1 Snapshot：歷史平均計算

### Q2: 有爆紅跡象嗎？（出圈/爆紅偵測）

**回答的核心問題**：哪些貼文有病毒傳播潛力

#### 分析參數

| 參數 | 設定 |
|------|------|
| 時間區間 | 近 72 小時內發布的貼文 |
| 資料範圍 | 單篇貼文篩選（符合條件的列表） |
| 適用層級 | Post |

#### 所需指標

| 指標 | 用途 |
|------|------|
| Virality Score | 綜合傳播潛力 |
| Early Velocity | 早期互動速度 |
| Growth Multiple | 曝光成長倍數 |
| Repost Rate / Quote Rate | 擴散力 |

#### 呈現方式

```
┌─────────────────────────────────────────┐
│  🔥 有爆紅跡象嗎？                        │
├─────────────────────────────────────────┤
│  發現 2 則貼文有出圈潛力：                 │
│                                         │
│  1. [貼文預覽]                           │
│     • Virality Score：8.5（高）          │
│     • Repost Rate：2.3%（爆紅跡象）       │
│     • 成長倍數：4.2x                     │
│     → 查看詳情                           │
│                                         │
│  2. [貼文預覽]                           │
│     • Virality Score：6.2               │
│     • Quote Rate：1.8%                  │
│     → 查看詳情                           │
│                                         │
│  📌 MVP：事後分析呈現                     │
│  📌 後續：即時警報推送                    │
└─────────────────────────────────────────┘
```

#### 判斷標準

| 指標 | 閾值 | 意義 |
|------|------|------|
| Repost/Quote Rate | > 2% | 爆紅跡象 |
| Growth Multiple | > 3x | 小爆預警 |
| Growth Multiple | > 5x | 爆紅 |
| Early Velocity | > 0.05 | 有爆紅潛力 |

#### 資料來源

- L3 Current：當前成效
- L1 Snapshot：計算成長倍數、早期速度

### Q3: 粉絲在成長嗎？（受眾成長追蹤）

**回答的核心問題**：帳號整體成長健康度、哪些貼文帶來新粉絲

#### 分析參數

| 參數 | 設定 |
|------|------|
| 時間區間 | 7 天 / 30 天趨勢 |
| 資料範圍 | 帳號層級（非貼文聚合） |
| 適用層級 | Account |

#### 所需指標

| 指標 | 用途 |
|------|------|
| Follower Growth | 粉絲淨成長 |
| Follower Growth Rate | 成長率 |
| 成長趨勢 | 週/月趨勢圖 |

#### 呈現方式

```
┌─────────────────────────────────────────┐
│  📈 粉絲在成長嗎？                        │
├─────────────────────────────────────────┤
│  本週粉絲：+85（成長率 +2.3%）            │
│                                         │
│  [趨勢圖：過去 30 天粉絲變化]              │
│                                         │
│  成長高峰：                              │
│  • 1/8（週三）+32                        │
│    可能來自：[貼文預覽]                   │
│  • 1/5（週日）+28                        │
│    可能來自：[貼文預覽]                   │
│                                         │
│  💡 洞察：問答類貼文帶來最多新粉絲          │
└─────────────────────────────────────────┘
```

#### 資料來源

- L3 Current：`workspace_threads_accounts.current_followers`
- L1 Snapshot：`workspace_threads_account_insights`

### Q4: 什麼內容表現最好？（內容類型優化）

> **📌 預留功能**：待標籤功能實作後再開發

**回答的核心問題**：哪種內容類型互動最高，指導內容策略

#### 分析參數

| 參數 | 設定 |
|------|------|
| 時間區間 | 30 天（需足夠樣本數） |
| 資料範圍 | 多篇貼文按類型聚合 |
| 適用層級 | Post → Account |
| **狀態** | 📌 預留（待標籤功能） |

#### 所需指標

| 指標 | 用途 |
|------|------|
| Engagement Rate（按類型聚合） | 各類型平均互動率 |
| Reply Rate（按類型聚合） | 各類型對話引發力 |

#### 內容類型分類

| 類型 | 判斷依據 |
|------|----------|
| 純文字 | 無媒體附件 |
| 圖片 | 含圖片附件 |
| 影片 | 含影片附件 |
| 連結 | 含外部連結 |
| 輪播 | 多張圖片 |

#### 呈現方式

```
┌─────────────────────────────────────────┐
│  🎯 什麼內容表現最好？                    │
├─────────────────────────────────────────┤
│  內容類型比較（過去 30 天）：              │
│                                         │
│  [長條圖：各類型平均 ER]                  │
│                                         │
│  排名：                                  │
│  1. 圖片貼文  平均 ER 7.2%  ← 最佳       │
│  2. 純文字    平均 ER 5.8%               │
│  3. 影片      平均 ER 4.5%               │
│  4. 連結      平均 ER 3.2%               │
│                                         │
│  💡 建議：圖片貼文互動率最高，可增加比例    │
└─────────────────────────────────────────┘
```

#### 資料來源

- L3 Current：貼文成效 + 貼文類型
- 需要：貼文類型欄位（`media_type`）

### Q5: 什麼時候發文最好？（最佳發文時段）

**回答的核心問題**：哪個時段發文互動最高

#### 分析參數

| 參數 | 設定 |
|------|------|
| 時間區間 | 30 天（需足夠樣本數） |
| 資料範圍 | 多篇貼文按時段聚合 |
| 適用層級 | Post → Account |

#### 所需指標

| 指標 | 用途 |
|------|------|
| Engagement Rate（按時段聚合） | 各時段平均互動率 |
| 發文數量（按時段） | 樣本數參考 |

#### 呈現方式

```
┌─────────────────────────────────────────┐
│  ⏰ 什麼時候發文最好？                    │
├─────────────────────────────────────────┤
│  最佳發文時段（根據你的歷史數據）：         │
│                                         │
│  [熱力圖：星期 x 時段]                    │
│                                         │
│  Top 3 時段：                            │
│  1. 週三 20:00-21:00  平均 ER 8.2%       │
│  2. 週日 10:00-11:00  平均 ER 7.5%       │
│  3. 週五 19:00-20:00  平均 ER 6.8%       │
│                                         │
│  ⚠️ 避免時段：                           │
│  • 週一 09:00-10:00  平均 ER 2.1%        │
│                                         │
│  📊 樣本數：過去 30 天共 45 則貼文         │
└─────────────────────────────────────────┘
```

#### 資料來源

- L3 Current：貼文成效 + 發布時間（`published_at`）
- 聚合計算：按星期 + 小時分組

### Q6: 哪些舊文還有流量？（長尾貼文追蹤）

**回答的核心問題**：識別常青內容、發現舊文爆發

#### 分析參數

| 參數 | 設定 |
|------|------|
| 時間區間 | 發布超過 30 天的貼文 |
| 資料範圍 | 單篇貼文篩選（符合條件的列表） |
| 適用層級 | Post |

#### 所需指標

| 指標 | 用途 |
|------|------|
| Views（近期成長） | 舊文是否仍有流量 |
| Growth Multiple | 舊文是否突然爆發 |
| 發布時間 | 判斷貼文年齡 |

#### 三種情境

| 情境 | 定義 | MVP 狀態 |
|------|------|----------|
| 常青內容 | 發布 > 30 天，近期仍有穩定成長 | ✅ 列表顯示 |
| 舊文爆發 | 舊貼文突然 Growth Multiple > 3x | ✅ 列表顯示（📌 即時警報放後續） |
| 生命週期 | 貼文流量隨時間的衰減曲線 | ✅ 圖表視覺化 |

#### 呈現方式

```
┌─────────────────────────────────────────┐
│  📚 哪些舊文還有流量？                    │
├─────────────────────────────────────────┤
│  常青內容（發布超過 30 天仍有流量）：       │
│                                         │
│  1. [貼文預覽] 發布於 2 個月前            │
│     本週 Views +1,234                   │
│     → 查看詳情                          │
│                                         │
│  2. [貼文預覽] 發布於 45 天前             │
│     本週 Views +567                     │
│     → 查看詳情                          │
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  近期爆發的舊文：                         │
│                                         │
│  🔥 [貼文預覽] 發布於 3 週前              │
│     成長倍數：4.5x（過去 24 小時）        │
│     → 查看詳情                          │
│                                         │
│  ─────────────────────────────────────  │
│                                         │
│  [圖表：內容生命週期比較]                  │
│  顯示多篇貼文的流量衰減曲線對比            │
└─────────────────────────────────────────┘
```

#### 資料來源

- L3 Current：當前成效
- L1 Snapshot：計算成長趨勢
- 篩選條件：`published_at < 30 天前`

## 頁面結構

```
Insight 頁面
├── Q1: 這週表現如何？        ✅ MVP
├── Q2: 有爆紅跡象嗎？        ✅ MVP
├── Q3: 粉絲在成長嗎？        ✅ MVP
├── Q4: 什麼內容表現最好？    📌 預留
├── Q5: 什麼時候發文最好？    ✅ MVP
└── Q6: 哪些舊文還有流量？    ✅ MVP
```

### 呈現方式選項

| 方式 | 說明 |
|------|------|
| **單頁滾動** | 所有問題在同一頁，上下滾動 |
| **Tab 切換** | 每個問題一個 Tab |
| **卡片摺疊** | 預設顯示結論，可展開詳情 |

**建議**：採用「卡片摺疊」，預設顯示結論，用戶可展開看詳細數據。

---

## 與總覽頁的差異

| 頁面 | 定位 | 呈現方式 |
|------|------|----------|
| **總覽頁** | 數據儀表板 | 顯示指標、圖表，讓用戶自己探索 |
| **Insight 頁** | 問題解答 | 直接回答問題，給出結論與建議 |
